SOP — Configuración de clientes

Procedimiento operativo

Dashboard Meta Ads

Cómo crear la API, configurar Supabase y agregar un cliente nuevo

Versión 1.0 · Equipo interno · GA Dashboard Meta

Resumen: Meta (token) → Supabase (base de datos) → Vercel (hosting) → /settings (cargar credenciales) → Listo para reportes.
1. App Meta→ 2. Token→ 3. Supabase→ 4. Vercel→ 5. /settings→ 6. Reportes ✓

Parte A — Crear la API en Meta

A1. Crear la aplicación

  1. Entrá a developers.facebook.com → Mis apps → Crear app.
  2. Tipo: App de negocio (u opción con Marketing API).
  3. Agregá el producto Marketing API a la app.

A2. Permisos del token

Permiso¿Cuándo?
ads_readObligatorio — ver reportes en el dashboard
ads_managementSolo si usan la sección Campañas (crear ads por API)

A3. Generar el token

  1. Abrí el Graph API Explorer (developers.facebook.com/tools/explorer).
  2. Seleccioná tu app y marcá los permisos ads_read.
  3. Generá el token. Debe empezar con EAA… sin comillas ni espacios.
  4. Probá: GET /me/adaccounts?fields=name,account_id — debe listar la cuenta del cliente.
Limitaciones: Si la app está en modo Desarrollo, solo usuarios con rol en la app ven datos. Sin Business Manager verificado: reportes ✅ · crear anuncios por API ❌ (usar Ads Manager).

Parte B — Supabase (una sola vez)

  1. Creá un proyecto en supabase.com.
  2. SQL Editor → ejecutá el archivo supabase/schema.sql del repo.
  3. Si la tabla ya existía, ejecutá también migration_page_id.sql y migration_widgets.sql.
  4. Copiá de Settings → API: URL y clave service_role (secreta).

Cada cliente guarda una fila en dashboard_settings (clave: client_slug). No hace falta crear filas manualmente.

Parte C — Vercel (hosting)

VariableObligatoriaDescripción
CLIENT_IDSíID del cliente (ej. go-apple)
ADMIN_PASSWORDSíContraseña para entrar a /settings
SUPABASE_URLSí*URL del proyecto Supabase
SUPABASE_SERVICE_ROLE_KEYSí*Clave service_role (secreta)
DASHBOARD_PASSWORDNoProteger la vista del cliente en /

* Necesarias para guardar desde la web. Sin Supabase solo funcionan variables META_* en env.

Parte D — Configurar desde la web

  1. Abrí tu-dominio.vercel.app/settings
  2. Ingresá la ADMIN_PASSWORD
  3. Pegá el token de Meta (EAA…)
  4. Elegí la cuenta de anuncios y la página del listado
  5. (Opcional) Contraseña para que el cliente entre al dashboard
  6. Guardar configuración
  7. Verificá en / que carguen los reportes del mes
Personalizar (admin): En / → botón ✎ Personalizar → widgets, metas mensuales por mes, y opción de ocultar Campañas.

Parte E — Agregar un cliente nuevo

  • Copiar clients/go-apple.json → clients/nuevo-cliente.json
  • Logo en public/clients/nuevo-cliente/logo.svg
  • Registrar en src/lib/clients.ts
  • Deploy en Vercel con CLIENT_ID=nuevo-cliente (o multi-cliente en un deploy)
  • Entrar a /settings y guardar el token de ese cliente
  • Probar reportes en /

Ejemplo de archivo cliente

{
  "id": "mi-cliente",
  "name": "Nombre Comercial",
  "tagline": "Reportes de Meta Ads",
  "logo": "/clients/mi-cliente/logo.svg",
  "primaryColor": "#0071e3",
  "accentColor": "#1d1d1f",
  "currency": "ARS",
  "locale": "es-AR",
  "timezone": "America/Argentina/Buenos_Aires",
  "defaultCountries": ["AR"]
}

Registrar en código (src/lib/clients.ts)

import miCliente from "../../clients/mi-cliente.json";

const CLIENTS = {
  "go-apple": goApple as ClientConfig,
  "mi-cliente": miCliente as ClientConfig,
};

Parte F — Desarrollo local

npm install
cp .env.example .env.local
# Completar .env.local
npm run dev

Si falla con error 500: npm run dev:clean (borra caché .next)

Solución de problemas

ProblemaSolución
API no configuradaToken o cuenta vacíos en /settings
Token inválidoRegenerar en Meta, pegar sin comillas
No guarda en webRevisar SUPABASE_URL y SERVICE_ROLE_KEY en Vercel
Error 500 en localnpm run dev:clean
No crea anunciosBM no verificado → Ads Manager + ocultar Campañas en Opciones
